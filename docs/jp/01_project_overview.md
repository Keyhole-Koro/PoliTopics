# 1. プロジェクト概要
[English Version](../../docs/01_project_overview.md)

## サービスの目的
PoliTopics は、日本の国会会議録を検索可能な要約と公開Web体験に変換する3部構成のパイプラインです。会議録を取り込み、LLMプロンプトタスクを構築し、要約を生成して、Web UIとAPIを通じて提供します。

## 解決する問題
- 国会会議録は長大で、主要なトピックや関係者をスキャンするのが困難である。
- 生データはトピック、人物、日付を横断して迅速に検索できるように構造化されていない。
- 重いペイロード（要約、対話）は、単一のクエリフレンドリーなストアに効率的に保存するには大きすぎる。

## ターゲットユーザー
- 国会審議の簡潔な要約を求める一般市民。
- トピックや人物ごとに会議を検索する必要があるジャーナリストや研究者。
- 取り込みと要約のパイプラインを実行する内部オペレーター。

## 用語集（ドメイン用語）
- 国会 API (National Diet API): 会議録を取得するために使用される外部 API (国会会議録検索システム)。
- 会議 (Meeting): 国会会議録 (issueID、院、日付、発言)。
- Issue ID: 会議録の一意の識別子 (タスクの PK として使用)。
- LLM タスク (LLM task): 会議の要約ジョブを表す DynamoDB レコード。
- チャンク (Chunk): チャンク要約のために長い会議プロンプトを分割したもの。
- リデュースプロンプト (Reduce prompt): チャンクの結果を組み合わせて記事にする最終プロンプト。
- 記事 (Article): DynamoDB + S3 に保存される要約された出力。
- 記事アセット (Article asset): S3 に保存され、URL で参照される大きなフィールド（要約、対話など）。
- 記事アセット URL (Article Asset URL): フロントエンドから記事アセットに安全にアクセスするために使用される、短命の署名付き URL。
- シンインデックス (Thin index): ファセットによる高速リスト/検索に使用される DynamoDB のセカンダリアイテム。
- LocalStack: ローカル開発に使用されるローカル AWS エミュレーター。
