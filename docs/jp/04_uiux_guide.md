# 4. UI/UX ガイド
[English Version](../../docs/04_uiux_guide.md)

## デザインルール
- 長文コンテンツの可読性のためのクリーンで高コントラストなレイアウト。
- 主要なセクションにはセリフスタイルを使用した大きく明確な見出し。
- 要約、参加者、キーワードを区分するためにカードとバッジを使用。

## タイポグラフィ
- フォントは Next.js 経由でロードされます: Inter と Manrope。
- デフォルトの本文テキストは `font-sans` (Inter) を使用します。
- 見出しは `font-serif` を使用することが多いです (Manrope が設定されていますが、オーバーライドされない限り実際のセリフフォントは Tailwind のデフォルトです)。

## カラーパレット (`frontend/app/globals.css` より)
- プライマリ: シアン (`--primary`, oklch(0.5 0.15 200))
- アクセント: ピンク (`--accent`, oklch(0.65 0.2 330))
- 背景: 白 (`--background`)
- 前景: グレー (`--foreground`)
- Muted/secondary: ライトグレー (`--muted`, `--secondary`)
- Destructive: 赤 (`--destructive`)

## コンポーネント仕様
- ボタン: `default`, `outline`, `ghost` バリアントを持つ `Button` コンポーネント。
- 入力: 検索用の大きな高さと `--ring` に一致するフォーカスリングを持つ `Input`。
- セレクト: カテゴリ/院/会議フィルタ用のリストを持つ `Select`。
- 日付ピッカー: 日付範囲選択用の `Popover` 内の `Calendar`。
- カード: 要約、参加者、キーワード、用語、対話に使用される `Card`。
- バッジ: カテゴリとキーワードに使用される `Badge`。
- 対話ビューア: 記事詳細ビュー内のフィルタ可能な議事録リスト。

## レスポンシブのルール
- ホームレイアウトは、グリッドレイアウトのために Tailwind のブレークポイント (`sm`, `md`, `lg`) を使用します。
- 記事詳細は `max-w-4xl` で中央揃えされ、モバイル用のパディングがあります。
- フィルタパネルは高さのトランジションを伴って折りたたみ/展開します。
