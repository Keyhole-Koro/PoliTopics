# 1. プロジェクト概要
[English Version](../../docs/01_project_overview.md)

## サービスの目的
PoliTopics は、人間による恣意的な操作を挟まず、日本の国会会議録を検索可能な要約と公開 Web 体験に変換するサービス。会議録を取り込み、LLM プロンプトタスクを構築し、要約を生成して Web UI と API で提供する。

## 解決する問題
- 国会会議録は長大で、主要なトピックや関係者をスキャンするのが困難。
- 生データはトピック、人物、日付を横断して迅速に検索できる形に整理されていない。
- 元の国会議事録 API は UI もドキュメントも扱いづらく、一般利用者には敷居が高い。

## ターゲットユーザー
- 国会審議の簡潔な要約を求める一般市民。

## 用語集（ドメイン用語）
- 国会 API (National Diet API): 会議録を取得するために使用される外部 API (国会会議録検索システム)。
- 会議 (Meeting): 国会会議録 (issueID、院、日付、発言)。
- Issue ID: 国会会議録 API の会議ID（グローバル一意ではない。会議メタデータとして保持）。
- Internal Task ID: `(session + house + issueID)` から導出する内部UID（DynamoDBタスクPK）。
- LLM タスク (LLM task): 会議の要約ジョブを表す DynamoDB レコード。
- チャンク (Chunk): チャンク要約のために長い会議プロンプトを分割したもの。
- リデュースプロンプト (Reduce prompt): チャンクの結果を組み合わせて記事にする最終プロンプト。
- 記事 (Article): DynamoDB + R2 に保存される要約された出力。
- 記事アセット (Article asset): R2 に保存され、URL で参照される大きなフィールド（要約、対話など）。
- 記事アセット URL (Article Asset URL): フロントエンドから記事アセットにアクセスするための短命の署名付きまたは公開 URL。
- シンインデックス (Thin index): ファセットによる高速リスト/検索に使用される DynamoDB のセカンダリアイテム。
- LocalStack: ローカル開発に使用されるローカル AWS エミュレーター。
