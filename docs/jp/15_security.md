# 15. セキュリティ設計
[English Version](../../docs/15_security.md)

## 認証
- DataCollection `/run` は `RUN_API_KEY` と一致する `x-api-key` を必要とします。
- Web API エンドポイントはパブリックです (コード内に認証なし)。

## 認可
- ロールベースのアクセス制御は実装されていません。

## シークレットの取り扱い
- `GEMINI_API_KEY` と `RUN_API_KEY` は環境変数または Terraform 変数を介して提供する必要があります。
- シークレットはリポジトリにコミットしてはいけません。

## CSRF / XSS
- Web API は GET ルートのみを公開しています。Cookie ベースの認証は使用されていないため、CSRF は現在関連ありません。
- フロントエンドは React ベースです。新しい HTML レンダリングを行う場合は、サニタイズされていることを確認してください。

## 脆弱性対応
- リポジトリ内に明示的なポリシーはありません (TBD)。パッチと開示のワークフローを文書化することを検討してください。
